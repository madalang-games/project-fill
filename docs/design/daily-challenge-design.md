# 데일리 챌린지 기획서 (Daily Challenge Design)

## 1. 개요

데일리 챌린지는 매일 단 하나의 특별 퍼즐을 전 세계 플레이어에게 동시 공개하여 공통 소재로 경쟁하는 소셜 루프의 핵심 기둥입니다. "오늘의 퍼즐"이라는 공유 경험은 글로벌 커뮤니티를 단일 이벤트로 묶고, 스트릭 보상으로 일일 접속 습관을 형성합니다.

---

## 2. 핵심 규칙

| 항목 | 내용 |
| :--- | :--- |
| **공개 주기** | 매일 00:00 UTC (고정) |
| **퍼즐 공유** | 전 세계 모든 플레이어가 동일한 퍼즐을 풀이 (시드 or 큐레이션) |
| **플레이 윈도우** | 24시간 (00:00 UTC 공개 → 다음날 00:00 UTC 만료) |
| **플레이 횟수** | 1회 (클리어 또는 포기 후 당일 재도전 불가) |
| **난이도** | Ch.3 수준 고정 (Signal Type 5–6개, 빈 레인 1–2개, 기믹 없음 or 단일 기믹) |
| **부스터** | 인게임 부스터(Shuffle/Add Lane/Undo) 모두 사용 가능, 골드 비용 동일 |

---

## 3. 보상 구조

### 3.1. 클리어 보상
- **기본 보상**: 100 골드 (클리어 시 1회 지급, 재도전 없으므로 중복 없음)
- **이동 횟수 기반 추가 보상 없음** (이동 횟수는 랭킹에만 활용)

### 3.2. 스트릭 보너스 (연속 클리어)

| 연속 클리어 | 추가 보상 |
| :---: | :--- |
| 3일 연속 | +50 골드 |
| 7일 연속 | +200 골드 + Shuffle ×3 |
| 30일 연속 | +500 골드 + **챌린지 전용 Chip Skin** (한정) |
| 100일 연속 | +1,000 골드 + **챌린지 전용 Board Skin** (한정) |

- 스트릭은 당일 챌린지 클리어 시에만 유지됩니다. 포기 또는 미참여 시 스트릭이 초기화됩니다.
- 스트릭 달성 시 별도 팝업으로 알림합니다.

---

## 4. 글로벌 랭킹

- 랭킹 기준: `moves_used` 오름차순 (이동 횟수가 적을수록 상위)
- 동점 시: `clear_time_seconds` 오름차순 (빠르게 클리어한 플레이어 상위)
- 노출: 랭킹 탭 → **챌린지(Challenge)** 탭
- 보관: 당일 랭킹은 24시간 후 아카이빙. 플레이어는 자신의 해당일 기록과 순위를 영구 조회 가능.

### 4.1. 내 챌린지 기록 요약

```
┌───────────────────────────────────────┐
│  오늘의 챌린지  2026-06-16            │
│  이동 횟수: 23회  |  순위: #142        │
│  스트릭: 🔥 7일 연속                   │
│                                       │
│  [결과 보기]  [오늘의 챌린지 하기]      │
└───────────────────────────────────────┘
```

## 5. UI 흐름

### 5.1. 챌린지 진입
- 로비 홈 탭 좌측 배지 영역: **챌린지 배지** (매일 갱신, 미완료 시 Pulse 애니메이션)
- 탭 시: 챌린지 정보 팝업 노출

```
┌────────────────────────────────────┐
│  📡  오늘의 챌린지                  │
│  #0147 — 2026.06.16               │
│  예상 난이도: ★★★☆☆                │
│  참여자: 1,204명                    │
│  🔥 스트릭: 7일 연속                │
│  ──────────────────────────────── │
│           [챌린지 시작]             │
│       [오늘의 랭킹 보기]            │
└────────────────────────────────────┘
```

### 5.2. 챌린지 완료
- 일반 ResultOverlay와 동일 UI 사용
- 추가로 **챌린지 결과 배너**: 이동 횟수, 현재 순위, 스트릭 상태 표시
- 스트릭 달성 시 별도 보상 팝업(RewardPopup) 연동

---

## 6. 퍼즐 생성 정책

| 방식 | 내용 |
| :--- | :--- |
| **큐레이션 우선** | QA가 검증한 퍼즐을 사전 등록. 관리자 CMS에서 날짜별 배정. |
| **절차적 생성 폴백** | 큐레이션 퍼즐 소진 시, 역방향 경로 알고리즘(Reverse-Path)으로 생성 + Solver 검증 후 자동 배정. |
| **시드 공유** | 동일 날짜의 퍼즐은 서버 시드 고정으로 전 세계 동일 배치 보장. |

## 7. 데이터 및 DB

### 7.1. `daily_challenges` 테이블 (서버)

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `challenge_date` | date (PK) | 챌린지 날짜 (UTC) |
| `stage_seed` | string | 퍼즐 생성 시드 or 큐레이션 ID |
| `signal_type_count` | int | Signal Type 수 |
| `lane_count` | int | Slot Lane 수 |
| `gimmick_id` | int? | 적용 기믹 (null = 없음) |

### 7.2. `user_daily_challenge_records` 테이블

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `user_id` | FK | 플레이어 |
| `challenge_date` | date | 챌린지 날짜 |
| `moves_used` | int | 사용 이동 횟수 |
| `clear_time_seconds` | int | 클리어 소요 시간 |
| `is_cleared` | bool | 클리어 여부 |
| `streak_at_clear` | int | 클리어 시점 연속 일수 |
| `reward_claimed` | bool | 보상 수령 여부 |

### 7.3. `user_challenge_streaks` 테이블

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `user_id` | FK (PK) | 플레이어 |
| `current_streak` | int | 현재 연속 클리어 일수 |
| `best_streak` | int | 역대 최고 연속 기록 |
| `last_clear_date` | date | 마지막 클리어 날짜 |

---

## 8. API 규약

| 메서드 | 경로 | 설명 |
| :--- | :--- | :--- |
| `GET` | `/api/daily-challenge/today` | 오늘의 챌린지 퍼즐 정보 + 내 참여 여부 |
| `POST` | `/api/daily-challenge/today/attempt` | 어테스트 시작 (stage attempt와 동일 패턴) |
| `POST` | `/api/daily-challenge/today/clear` | 클리어 결과 제출 |
| `GET` | `/api/daily-challenge/today/ranking` | 오늘의 글로벌 랭킹 (페이지네이션) |
| `GET` | `/api/daily-challenge/today/me` | 내 오늘 기록 및 순위 |
| `GET` | `/api/daily-challenge/streak` | 내 스트릭 정보 |
