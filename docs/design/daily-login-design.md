# 일일 출석 시스템 기획서 (Daily Login / Attendance System Design)

## 1. 개요

일일 출석 시스템은 플레이어의 일일 접속 습관을 보상하는 가장 기본적인 리텐션 루프입니다. 페널티 없는 관대한 정책(비연속도 사이클 유지)을 채택하여 릴랙싱 퍼즐의 정체성과 충돌 없이 접속 동기를 부여합니다.

---

## 2. 사이클 정책

| 항목 | 내용 |
| :--- | :--- |
| **사이클 길이** | 7일 (Day 1~7, 완주 후 Day 1 재시작) |
| **사이클 유지 조건** | 비연속 허용 — 오늘 출석하면 사이클이 중단되지 않고 이어서 진행 |
| **스트릭 카운트** | 별도 유지. 연속 접속일 수가 끊기면 스트릭 초기화 (사이클 자체는 유지) |
| **중복 수령 방지** | 당일 UTC 기준 최초 1회만 보상 지급 |

> **설계 의도**: 사이클을 리셋하지 않으므로 며칠 빠진 유저도 "진도를 잃었다"는 느낌 없이 복귀 가능. 단, 스트릭이 끊기므로 연속 참여 동기는 유지됨.

## 4. UI

### 4.1. 출석 팝업

- **진입**: 매일 첫 로그인 시 자동 팝업 (로비 홈 탭 진입 직후).
- **레이아웃**: 7개 Day 카드 가로 나열, 현재 Day 강조 (Pulse 테두리).

```
┌───────────────────────────────────────────┐
│  📅  오늘의 출석 보상                       │
│                                           │
│  [✓] [✓] [✓] [ ] [ ] [ ] [🎁]            │
│  D1  D2  D3  D4  D5  D6  D7              │
│                                           │
│  오늘 보상: 🪙 100 골드                    │
│                                           │
│           [보상 받기]                      │
└───────────────────────────────────────────┘
```

- 완료된 Day: 체크 아이콘 + 어두운 오버레이.
- 오늘 Day: 펄스 강조 + 보상 미리보기.
- 이미 수령한 경우: 팝업 자동 스킵 (재진입 시 배지만 표시).

### 4.2. 홈 탭 배지

- IAP Badge Design의 `EventLayoutGroup` 좌측에 배치.
- 미수령 시: 보상 아이콘 + "출석 보상" 텍스트 + 레드 도트(알림).
- 수령 후: 배지 비활성화 (다음날 00:00 UTC 갱신).

## 5. 데이터 및 DB

### 5.1. `user_login_attendance` 테이블

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `user_id` | FK (PK) | 플레이어 |
| `current_day` | int | 현재 사이클 내 Day (1~7) |
| `current_cycle` | int | 현재 사이클 번호 (1-based) |
| `current_streak` | int | 현재 연속 접속 일수 |
| `best_streak` | int | 역대 최고 연속 접속 |
| `total_attended_days` | int | 누적 출석 일수 (비연속 포함) |
| `last_attended_date` | date | 마지막 출석 날짜 (UTC) |
| `reward_claimed_today` | bool | 오늘 보상 수령 여부 |
