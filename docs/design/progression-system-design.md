# 진행 시스템 기획서 (Progression System Design)

## 1. 개요
진행 시스템은 플레이어가 게임을 진행하는 방식, 이동 횟수(Moves)를 통한 성적 기록, 그리고 챕터(Chapter) 마일스톤을 통한 장기 목표 달성을 정의합니다.

---

## 2. 이동 횟수 기록 (Moves)
모든 스테이지는 클리어 시 이동 횟수를 기록합니다. 이동 횟수는 보상에 영향을 주지 않으며 랭킹 산정에만 사용됩니다.

| 항목 | 내용 |
| :--- | :--- |
| `latest_moves` | 가장 최근 클리어의 이동 횟수 |
| `best_moves` | 해당 스테이지 최소 이동 횟수 기록 |
| 랭킹 기준 | `best_moves` 오름차순 (적을수록 상위) |
| 보상 영향 | 없음 (이동 횟수 무관 고정 보상) |

### 실패 조건
- Signal Sort는 턴 제한이 없으므로 시간이나 턴 수 소진으로 실패하지 않습니다.
- 더 이상 이동할 칩이 없는 교착 상태(Stuck)에서 부스터나 광고 시청 구제를 거부하고 **직접 로비로 퇴장(포기)하거나 스테이지 재시작**을 선택할 경우 클리어 실패로 기록됩니다.

---

## 3. 챕터 구조
스테이지는 테마별 템포 조절 및 보상 단위를 위해 챕터 단위로 구성됩니다.

### 챕터 속성
- **챕터 ID:** 순차적인 그룹화 ID.
- **스테이지 수:** 챕터당 20스테이지 고정.
- **테마 비주얼:** 각 챕터는 고유한 `theme_key`(배경 테마, 칩 스킨, 컬러 팔레트)를 가집니다.

---

## 4. 마일스톤 보상 (챕터 완료)
챕터 내 모든 스테이지를 최초 클리어하면 챕터 완료 보상이 지급됩니다.

### 챕터 완료 조건 및 보상
- **조건:** 챕터 내 모든 스테이지 **최초 클리어**. 이동 횟수 및 부스터 사용 여부 무관.
- **상호작용:** 조건이 충족되면 챕터 선택 UI에 보상 상자가 활성화됩니다.
- **보상 소스 바인딩:** 상자는 CSV의 고유한 `reward_source`(예: `chapter1_chest`)에 매핑되며, 이는 서버의 `reward_group_id`로 해석됩니다.
- **서버 검증:** 서버는 `user_stage_progress` 테이블에서 해당 챕터의 모든 `stage_clear = true` 여부를 확인한 후 보상을 지급합니다.

| 챕터 | 골드 보상 | 부스터 보상 |
| :---: | :---: | :--- |
| Ch.1 | 500골드 | Shuffle ×3 |
| Ch.2 | 800골드 | Shuffle ×5, Add Lane ×1 |
| Ch.3 | 1,200골드 | Shuffle ×5, Add Lane ×2 |
| Ch.4 | 1,600골드 | Shuffle ×8, Add Lane ×3 |
| Ch.5 | 2,000골드 | Shuffle ×10, Add Lane ×5 |

---

## 5. 비주얼 테마 진행 (쉐이더 기반)

비주얼 테마는 커스텀 쉐이더(`UIPulseGlow.shader` 또는 테마 기반 쉐이더)와 머티리얼 인스턴스를 통해 구현됩니다.
- **테마 머티리얼**: 각 챕터는 시각적 요소와 색조(Tint) 속성을 교체하는 별도의 머티리얼에 매핑됩니다.
- **쉐이더 변수**: 쉐이더는 `_ThemeColor1`, `_ThemeColor2` 및 텍스처 마스크를 읽어 배경 패널, 슬롯 레인, 점등 효과의 색상을 동적으로 변경하여 애셋 중복을 방지합니다.
- **해결**: 활성 `chapter.bg_theme_id`와 일치하는 `VisualService`에 의해 런타임에 동적으로 로드됩니다.
