# 진행 시스템 기획서 (Progression System Design)

## 1. 개요
진행 시스템은 플레이어가 게임을 진행하는 방식, 별(Star)을 통한 성공 측정, 그리고 챕터(Chapter) 마일스톤을 통한 장기 목표 달성을 정의합니다.

---

## 2. 별 시스템 (성적 지표)
모든 스테이지에는 3단계의 성적 티어(별)가 있으며, 클리어 시점의 누적 이동 횟수(Moves)를 기준으로 성적이 결정됩니다.

### 별 획득 기준
| 티어 | 조건 | 정의 |
| :---: | --- | --- |
| **별 1개** | `MovesUsed > Star2_Moves` | **최소 클리어.** 스테이지 정렬을 완료하였으나 이동 횟수가 많음. |
| **별 2개** | `Star3_Moves < MovesUsed <= Star2_Moves` | **중급 달성.** 보드 정렬 숙련도 증명. |
| **별 3개** | `MovesUsed <= Star3_Moves` | **완벽 클리어.** 목표 이동 횟수(Star3_Moves) 이하로 최적화 정렬 성공. |

### 실패 조건 (별 0개)
- Signal Sort는 기본적으로 턴 제한이 없으므로 시간이나 턴 수 소진으로 실패하지 않습니다.
- 플레이 도중 더 이상 이동할 칩이 없는 교착 상태(Stuck)에 빠졌을 때, 부스터(셔플, 레인 추가, 되돌리기) 사용이나 광고 시청을 통한 구제를 거부하고 플레이어가 **직접 로비로 퇴장(포기)하거나 스테이지 재시작(Restart)**을 선택할 경우에 실패로 기록됩니다.

---

## 3. 챕터 구조
스테이지는 테마별 템포 조절 및 보상 단위를 위해 챕터 단위로 구성됩니다.

### 챕터 속성
- **챕터 ID:** 순차적인 그룹화 ID.
- **테마 비주얼:** 각 챕터는 고유한 `theme_key`(배경 테마, 칩 스킨, 컬러 팔레트)를 가집니다.
- **스테이지 범위:** 예: 챕터 1은 스테이지 1~20번을 포함합니다.

---

## 4. 마일스톤 보상 (챕터 완료)
재플레이 가치와 완벽주의를 장려하기 위해 챕터별 100% 완료 시 보상을 제공합니다.

### 챕터 완료 상자
- **조건:** 현재 챕터 내의 모든 스테이지에서 **별 3개**를 획득해야 합니다.
- **상호작용:** 조건이 충족되면 챕터 선택 UI에 "황금 상자"가 나타납니다.
- **보상:** 대량의 골드 및 고급 부스터 세트 (예: 1,500 골드, Undo 5개, Shuffle 3개, Hint 2개, Add Lane 1개). 스태미나 개념이 없으므로 무제한 스태미나 등의 상품은 보상에서 배제됩니다.
- **보상 소스 바인딩:** 상자는 CSV의 고유한 `reward_source`(예: `chapter1_chest`)에 매핑되며, 이는 서버의 `reward_group_id`로 해석됩니다.
- **서버 검증:** 서버는 보상을 지급하기 전에 `user_stage_progress` 테이블을 조회하여 챕터의 모든 스테이지가 실제로 별 3개를 획득했는지 검증합니다.

---

## 5. 비주얼 테마 진행 (쉐이더 기반)

비주얼 테마는 커스텀 쉐이더(`UIPulseGlow.shader` 또는 테마 기반 쉐이더)와 머티리얼 인스턴스를 통해 구현됩니다.
- **테마 머티리얼**: 각 챕터는 시각적 요소와 색조(Tint) 속성을 교체하는 별도의 머티리얼에 매핑됩니다.
- **쉐이더 변수**: 쉐이더는 `_ThemeColor1`, `_ThemeColor2` 및 텍스처 마스크를 읽어 배경 패널, 슬롯 레인, 점등 효과의 색상을 동적으로 변경하여 애셋 중복을 방지합니다.
- **해결**: 활성 `chapter.bg_theme_id`와 일치하는 `VisualService`에 의해 런타임에 동적으로 로드됩니다.
