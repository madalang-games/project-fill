# Scripts/InGame/View

Signal Sort board views. UI Chrome (HUD, Booster Bar, Stuck/Clear/Pause popups) is **authored on the InGame scene canvas via `UIEditorSetup.SetupInGame`**. The core gameplay board (Lanes, Chips, Signal Panel) is placed in **World Space** under `BoardWorldRoot`, managed by `BoardWorldResizer` matching Canvas screen constraints. Visuals stay procedural via `WorldUtil` using `SpriteRenderer` and `TextMeshPro` 3D.

## Files

Gameplay-fixed marker (not a cosmetic axis) → drawn straight from `TextureFactory`, not the `SpriteSet`. |
| `SignalNodeView.cs` | `SignalNodeView` | One World Space Signal Panel node: layered procedural LED — recessed dark socket `_rim`, disc `_body`, static glassy specular `_spec`, white-hot `_ring`, soft `_glow` halo, 3D glyph, connector line. **Idle** unlit nodes are tinted toward their signal color (`_idleBody/_idleRing/_idleGlyph`) so the panel previews which signals it tracks. **Lit (completed)** breathes a synced **dark↔bright neon pulse** (`Update` one smoothstepped `pulse`: body/ring lerp dim-neon↔hot-white via `Dim()`, glow + spec flare ride the same beat). |

## Rules
- **Hybrid Canvas/World Architecture**: Canvas owns UI Chrome (HUD, BoosterBar, Stuck/Clear/Pause popups). The gameplay board (Lanes, Chips, SignalPanel) lives in the 3D World Space scene under the `BoardWorldRoot` root object.
- **Aspect Sizing**: `BoardWorldResizer` dynamically moves and scales `BoardWorldRoot` and its children relative to `GameArea` Canvas bounds. Never hardcode absolute world bounds.
- **Visible board surface + chrome clearance**: Board is sized off the **camera viewport**, NOT GameArea (GameArea can exceed the screen → the old left/right overflow). `BoardWorldResizer` fits a themed surface into the **band between HUD (top) and BoosterBar (bottom)** (`_topBar`/`_bottomBar`, auto-found under `SafeAreaRoot`) so it never hides behind chrome. Serialized tunables: `_marginX` = left/right margin as a **fraction of viewport width**; `_innerPad` = pieces inset from the surface edge (small → bigger lanes/chips, easier taps); `_barGap` = gap off the bars; `_panelHeightFrac` = Signal Panel share. Surface via `BoardView.SetBoardSurface` (fill `Surface` + outline `SurfaceBorder`, **sortingOrder 0** = above background canvas, below pieces ≥1). **Lanes re-fit live** (`BoardView.ApplyLanesFit`; BuildLanes only caches `_gridW`/`_gridH`) so `_marginX`/`_innerPad` actually resize lanes, not just at build. Surface color follows the BOARD accent. Missing bars → band falls back to viewport edges.
- **Input Checking**: Raycasting is handled by `BoardView.Update()` checking click position on `LaneView` colliders. Input is ignored if `EventSystem.current.IsPointerOverGameObject()` is true to prevent UI overlaps (e.g. Pause button, Booster Bar) from clicking world lanes.
- `BuildLanes` tags each lane GO with a `TutorialTarget`: `slot_lane_{i+1}` + a gimmick id when the lane carries one (`gimmick_locked`/`gimmick_blind` by `SlotLane.Kind`, `gimmick_overload` if any chip is overload); also `slot_lane_area` (container) + `signal_panel` (`_panelContainer`). `SignalPanelView.Initialize` tags each node `signal_node_{i+1}` (i=0 = first in relay order) for the Relay tutorial. Lets `TutorialOverlay` spotlight real lanes/gimmicks/nodes by id. Rebuilt every Init (OnDisable/OnEnable re-registers).
- **Completed lane stays (spec A-R05)**: Emptied + reusable, never removed. Empty only after the sweep.
- `BuildLanes` lays lanes out as a **fixed-size grid**: lane W/H/gap are serialized constants (`_laneWidth`/`_laneHeight`/`_laneGap`), never stretched to fill. Columns wrap past `_maxColumns` into balanced rows (each row centered by its own count). The whole grid is then **uniformly shrunk** (never enlarged) to fit the container via `_lanesContainer.localScale` (`_lanesScale`). Flight FX spawn on the unscaled `_flightLayer`, so chip flight size is multiplied by `_lanesScale`.
- Blind/Pending markers sit above the lane frame (not over the top slot) so they never cover the revealed top chip.
- **Gimmick state is text-free (icon + FX only)** — no localized/hardcoded words, no emoji (avoids font-subset tofu): **Locked** = `TextureFactory.Padlock` (slot-in `BoardSkin.lockSeal`) tinted to `UnlockType` color (color = which signal opens it); unlock plays `UnlockRoutine` shrink-dissolve via `Refresh` lock→unlock transition. **Blind** = "?" disc badge + `_scanLine` sweep animated over the masked stack in `Update`. **Relay pending** = `_pips` queue dots (filled = `PendingNumber` steps until next; ≤1 = green "up next", clamps at `MaxPips`) + amber border pulse; order itself is read from the Signal Panel node sequence. **Overload** = `ChipView` corner lightning badge (`TextureFactory.Bolt` on a dark disc) + periodic charge→discharge GlowSweep (orange halo flash + body throb + in-bounds bright sweep bar); text-free, gameplay-fixed (not skinnable).
- Selection is highlight-only (glow/scale + border) — no vertical chip lift, so a batch pour starts evenly from slot positions. Select fires a one-shot scale **punch** (`ChipView._punch`) that decays into the steady glow/scale pulse.
- `TextureFactory.Glow` is a **soft radial** sprite (small bright core → transparent at the edge, NOT 9-sliced) so it reads as a compact halo; an oversized/hard glow blooms over neighbors. Tint-able white. Glow render size lives at the call site (proportional to node/chip size).
- **Signal Panel nodes are fixed LED-size**: `SignalPanelView` caps node diameter to `panelHeight * 0.42` (clamped to cell width), NOT stretched to fill the wide cell — keeps lit disc/ring/glow compact. `SignalNodeView` glow is `size * 1.2` and the register pop is a gentle blink (`*0.18`), not a swell.
- **`SignalNodeView` disc/ring scale must use the captured base scale** (`_bodyBase`/`_ringBase`), never `Vector3.one`: they are NON-9-sliced sprites, so `WorldUtil.CreateSprite` sets their `localScale` to fit `size` (≈0.3). Writing `Vector3.one` (old `PopRoutine` bug) blew the LED up ~3× into neighbors. State changes lerp colors toward targets (`_tBody`/`_tRing`/…) in `Update` for a smooth neon transition; register fires a decaying glow `_flash`. **Static depth**: a recessed dark `_rim` socket behind the disc + a static glassy `_spec` highlight (upper-left) make each node read as a seated domed bulb, not a flat sticker; **unlit nodes carry their signal color faintly** (`_idleBody/_idleRing/_idleGlyph` = dark base Lerp'd toward `Type.ToColor()`) so the panel previews its objectives at rest. **Lit (completed) nodes pulse a dark↔bright neon breathe**: one shared smoothstepped `pulse` (`Mathf.Sin`, slow ~2.4) drives body/ring between a dim-neon (`Dim()` = rgb×f, alpha kept opaque) and a hot near-white version of the signal color, with the `_glow` halo and `_spec` flare riding the same beat; the lit glyph recedes (`_tGlyph` faint alpha + `_glyphBase*0.7` shrink) so the colorblind legend stays without fighting the pulse. `Dim()` keeps alpha=1 (raw `Color*float` would fade the body transparent).
- Impact accents (all procedural sprite coroutines on `_flightLayer`, self-destroy; no Unity ParticleSystem):
  - **Chip landing** (`DropAll`): `SpawnRing` with `flattenY` (ground-hugging shockwave) + `SpawnMotes` (small upward dust cone, arcs down via `BurstParticle` gravity). Kept subtle — fires per chip, high frequency.
  - **Node register** (`CompleteSweep`): double `SpawnRing` (inner crisp + outer wide) + `SpawnShards` (rounded-square pixel "data bits" burst radially, spin + fall) + `SpawnTrace` (light pulse runs along the connector to the next node via `SignalPanelView.TryGetTraceTarget`, false for last node) + node `PlayRegister` glow. Climactic, low frequency.
  - `SpawnRing(...flattenY=1f)` squashes Y when <1; `BurstParticle` is shared (motes); `ShardParticle`/`TraceRoutine` are register-only.
- `LaneView.ShakeRoutine` (invalid-move) captures home position **live** at shake start, never a cached `_homePos` — Construct runs before BoardView assigns the lane position, so a cached home snaps the lane to the board origin (corrupts layout + tap colliders).
- **Concept: abstract "Signal Sort" — NOT skeuomorphic hardware.** The game is a color-sort stacker; identity is by COLOR (10 signals), so tokens stay shape-neutral and high-contrast. A lane = a **vertical signal-register column** holding stacked **glowing data cells**; the circuit/motherboard flavor is a SKIN palette only and is carried by the **background + Signal Panel + connector traces**, never by detailing the lane as a physical DIMM/PCIe slot (hardware texture on each lane kills color readability). Earlier RAM-slot props (contacts/insert-socket/ejectors/silkscreen) were removed for this reason.
- Art direction is **color-token + pixel/casual**: `ChipView` renders the cell body in the signal color with a bold outline **plus a top gloss `_sheen`** so the tinted square reads as a luminous data cell (hidden on blind cells); `SignalNodeView` lights as a glowing bulb (disc core + white-hot ring). Real sprites slotted into `BoardSkin`/`SpriteSet` drive the shape; these colors still tint them.
- **Board-skin cosmetic theming**: `BoardView.Awake` resolves `BoardTheme.Resolve(CosmeticState.ResolveBoardSkin(), ActiveChipSkin, ActiveLaneSkin)` into the `SpriteSet`; `ChipView`/`LaneView`/`SignalNodeView` read outline/fill/glow/border/socket/contacts colors from the bundle instead of hardcoded constants. **Never re-hardcode those** — add a token to `BoardTheme`/`SpriteSet` and a per-id value. The chip body hue MUST stay the signal color (gameplay identity); skins only restyle outline/fill-alpha/glow + lane/node frame + the cosmetic finish/FX axes: **`ChipFinish`** (procedural surface material composited over the body via `ChipView._finish`) plus the price-scaled FX tokens (`SurfaceEdgePulse`/`BoardMoteCount`/`SurfaceEdgeSparkCount`/`SpectrumCycle`/`LaneEdgePulse`/`ChipOutlinePulse`/`ChipGhost`/`ChipSpectrum`). Add a new finish by extending the `ChipFinish` enum + `TextureFactory.ChipFinishOverlay`; add/raise an FX token by adding a field to `BoardTheme`+`SpriteSet` + a per-id arm in `BoardTheme.Board`/`ApplyChip`/`ApplyLane` + the per-frame drive in the owning view's `Update` — never hardcode in the views. Board pieces build once at scene entry, so a cold direct-InGame entry (cosmetics unfetched) renders the default theme — acceptable fallback (normal lobby→InGame flow has `CosmeticState` populated). Background ambient theme is separate (`BoardBgTheme`/`InGameSceneBackgroundView`).
- **Lane structure + skin/state channel split**: `LaneView` border is the **interaction-state channel ONLY** (select=blue / valid=green pulse / invalid=red / pending=amber / blind=cyan) — `Update` overwrites it every frame, so lane skins must NOT carry identity there. Skin identity lives in the **inner backlight** (`_backlightColor`=`LaneBacklight`, board-accent by default, breathes on Dynamic) + the premium **`LaneEdgePulse`** breathe (backlight pulse + a tight inner accent frame `_edgeFrame`, inset to `x*0.94`/`y*0.97` so a breathing premium lane stays INSIDE its bounds and never bleeds onto neighbours). The lane is kept **minimal on purpose** — overdecoration (rails / gauge / silkscreen were all tried and cut) reads as amateur and steals contrast from the color chips, which are the star. The old vertical channel-flow sweep (`LaneFlow`) was removed: it travelled like the blind `_scanLine` and read as a blind marker on cyan lane skins — premium lane motion is now the in-bounds breathe (no vertical travel). Remaining structure: **continuous channel** (recessed track spanning all cells so the lane reads as ONE register column, not 4 boxes — keep socket guides faint so they don't fight it) + **cell separators** (thin dividers, color `_railColor`=`LaneRail`). Sorting: backlight 2 / body+channel 3 / cell guides 4 / separator+border+edgeFrame 5, chips ≥7 (always on top). Add new lane structure colors / FX as `BoardTheme`/`SpriteSet` tokens — never hardcode.
- Chip outline and lane outline are **separate `BoardSkin` slots**: `chipOutline` → `ChipView` border only; `laneOutline` → `LaneView` frame/sockets/insert. Both fall back to the same `RoundedOutline` but slot independently.
- Keep legacy stub views — removing `CellView`/`BoardView.GetCellView` breaks `Core/UI/TutorialOverlay`.
